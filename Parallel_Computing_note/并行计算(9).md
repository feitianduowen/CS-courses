## 2. 14.1 基于消息传递的编程

### 2.1 基本思想

在消息传递模型中，一个并行应用由一组进程组成。

每个进程：

```text
只能直接访问自己的私有数据
不能直接访问其他进程地址空间
通过 send / receive 传递消息
通过消息交换实现数据共享和同步
```

这和共享内存模型不同。共享内存中线程可以直接读写共享变量；消息传递中进程必须显式发送和接收数据。

------

### 2.2 消息传递库

课件列出了一些流行的消息传递软件，例如：

## 3. MPI 和 PVM 简介

### 3.1 MPI 简介

MPI 是 **Message Passing Interface**，消息传递接口。

课件给出的 MPI 发展过程包括：

```text
1992年4月：组建消息传递接口标准工作组
1992年10月：初稿形成，主要定义点对点通信接口
1993年1月：第一届 MPI 会议在 Dallas 举行
1994年5月：MPI 标准正式发布
1997年：MPI-2 发布，原来的 MPI 更名为 MPI-1
```

MPI 的目标是提供一个：

```text
实际可用
可移植
高效
灵活
```

的消息传递接口标准。

MPI 是语言无关的接口库，并提供 C、Fortran 和 Java 等语言绑定。

------

### 3.2 MPI 的实现

MPI 可以建立在厂家专用环境上，例如：

```text
IBM SP2 的 POE/MPL
Intel Paragon 的 OSF/Nx
```

也可以使用公共 MPI 环境，例如：

```text
CHIMP
LAN
MPICH
```

课件特别提到，MPICH 是 MPI 在多种机器上的可移植实现，可以安装在：

```text
PC
工作站
SMP
MPP
COW
```

------

### 3.3 PVM 简介

PVM 是 **Parallel Virtual Machine**，并行虚拟机。

课件给出的信息是：

```text
开发时间：始于1989年
开发单位：Tennessee大学、Oak Ridge国家实验室、Emory大学
特点：适应性好、可扩展、可移植、易使用、源码可免费获得
```

PVM 可以把多台异构计算机组织成一个“虚拟并行机”，让用户像使用一台并行机一样运行程序。

------

### 3.4 MPI 和 PVM 的主要差别

课件列出三点：

```text
1. PVM 是自包含系统，MPI 不是。
2. MPI 对消息传递提供更强大的支持。
3. PVM 不是标准，因此版本更新更灵活。
```

第一点最关键。

PVM 自己包含进程管理和 I/O 等功能；MPI 依赖底层平台提供进程管理和 I/O 能力。

不过两者后来功能逐渐趋同：MPI-2 增加了进程管理功能，PVM 也提供了更多群集通信函数。

## 4. 消息传递方式

### 4.1 通信模式需要理解的三个问题

课件第 11 页指出，理解通信模式时要关注：

```text
共有多少个进程？
进程间如何同步？
如何管理通信缓冲区？
```

消息传递系统常用三种通信模式：

```text
同步消息传递
阻塞消息传递
非阻塞消息传递
```

------

### 4.2 同步消息传递

同步通信要求发送方和接收方都到达对应的发送和接收点。

发送返回意味着：

```text
消息已经被收到
```

优点是语义明确，不需要临时系统缓冲区。

缺点是等待时间最大，通信和计算不能重叠。

------

### 4.3 阻塞消息传递

阻塞发送的开始条件是发送方到达发送点。

发送返回意味着：

```text
消息已经发送完
```

接收返回意味着：

```text
消息已经收到
```

它介于同步和非阻塞之间，可能需要系统提供临时缓冲区。

------

### 4.4 非阻塞消息传递

非阻塞发送返回只意味着：

```text
已经通知系统某个消息要被发送
```

非阻塞接收返回只意味着：

```text
已经通知系统某个消息要被接收
```

它可以让通信与计算重叠，但需要额外的状态检查，例如 `wait` 或 `test`。

------

### 4.5 非阻塞通信的额外开销

课件第 15 页指出，非阻塞模式也会带来额外开销：

```text
临时缓冲区内存空间
分配缓冲区操作
消息拷入和拷出临时缓冲区
额外的检测和等待函数
```

所以非阻塞通信不是“免费加速”。它只有在计算和通信能真正重叠时才有意义。

------

### 4.6 消息传递模型的优缺点

优点：

```text
用户可以完全控制并行性开发、数据分布和通信实现
```

缺点：

```text
通信位置、数据移动、数据复制、一致性都要程序员显式处理
科学计算中显式域分解很困难
不容易通过逐步修改串行程序实现并行化
```

这一点和 OpenMP 形成对比。OpenMP 可以在原串行代码中逐步插入并行指令，而 MPI 程序通常需要从数据分布开始整体重构。

## 6. MPI 中的消息

### 6.1 MPI 程序基本结构

课件给出一个计算：

```text
Σ foo(i)
```

的 MPI SPMD 程序。

核心函数包括：

```c
MPI_Init(&argc, &argv);
MPI_Comm_size(MPI_COMM_WORLD, &group_size);
MPI_Comm_rank(MPI_COMM_WORLD, &my_rank);
MPI_Send(...);
MPI_Recv(...);
MPI_Finalize();
```

含义分别是：

| 函数          | 作用             |
| ------------- | ---------------- |
| MPI_Init      | 初始化 MPI 环境  |
| MPI_Comm_size | 获取进程组大小   |
| MPI_Comm_rank | 获取当前进程编号 |
| MPI_Send      | 发送消息         |
| MPI_Recv      | 接收消息         |
| MPI_Finalize  | 结束 MPI 环境    |

------

### 6.2 MPI 进程的特点

课件强调：

```text
MPI进程是重型的单线进程。
它们拥有不同的地址空间。
一个进程不能直接访问另一个进程地址空间中的变量。
```

因此，进程间通信必须靠消息传递实现。

------

### 6.3 MPI 消息的组成

课件把 MPI 消息分为两部分：

```text
消息缓冲 message buffer
消息信封 message envelope
```

消息缓冲是“信的内容”，即真正要发送的数据。

消息信封是“信封上的地址信息”，说明消息发给谁、用什么标签、属于哪个通信子。

------

### 6.4 MPI_Send 的参数

典型发送调用：

```c
MPI_Send(&N, 1, MPI_INT, i, i, MPI_COMM_WORLD)
```

一般形式：

```c
MPI_Send(buffer, count, datatype, destination, tag, communicator)
```

其中：

```text
buffer, count, datatype 定义消息缓冲
destination, tag, communicator 定义消息信封
```

也就是说，MPI 不只是发送一个地址，还必须说明数据项数量、数据类型、目的进程、标签和通信范围。

------

## 7. MPI 数据类型

### 7.1 为什么需要消息数据类型

课件给出两个原因：

```text
支持异构计算
允许发送非连续、非均匀内存区中的消息
```

异构计算中，不同机器可能使用不同的数据表示方式。MPI 数据类型能帮助系统在不同平台之间正确解释数据。

------

### 7.2 MPI 预定义数据类型

课件列出一些常见预定义类型：

| MPI 类型     | C 类型       |
| ------------ | ------------ |
| MPI_CHAR     | signed char  |
| MPI_DOUBLE   | double       |
| MPI_FLOAT    | float        |
| MPI_INT      | int          |
| MPI_LONG     | long         |
| MPI_SHORT    | short        |
| MPI_UNSIGNED | unsigned int |
| MPI_PACKED   | 打包数据     |

Fortran 中也有对应类型，例如：

```text
MPI_INTEGER
MPI_REAL
MPI_DOUBLE_PRECISION
MPI_CHARACTER
MPI_LOGICAL
```

------

### 7.3 非连续数据和导出数据类型

如果要发送数组中的偶数编号元素：

```text
A[0], A[2], A[4], ...
```

这些元素在内存中不是连续存放的。普通的：

```text
(address, count, datatype)
```

无法直接描述这种消息。

MPI 提供 **derived datatype**，即导出数据类型。

例如课件用：

```c
MPI_Type_vector(50, 1, 2, MPI_DOUBLE, &EvenElements);
MPI_Type_commit(&EvenElements);
MPI_Send(A, 1, EvenElements, destination, ...);
```

表示从数组 `A` 中每隔一个元素取一个双精度数，共取 50 个。

------

### 7.4 MPI_Type_vector 的含义

形式是：

```c
MPI_Type_vector(count, blocklength, stride, oldtype, &newtype)
```

含义：

```text
count：块的数量
blocklength：每块中连续 oldtype 数据项数量
stride：相邻块之间相隔多少个 oldtype 元素
oldtype：已有类型
newtype：构造出的新类型
```

例如：

```c
MPI_Type_vector(50, 1, 2, MPI_DOUBLE, &EvenElements)
```

表示：

```text
50 个块
每块 1 个 double
相邻块间隔 2 个 double
```

因此正好取出偶数位置元素。

## 8. 消息缓冲

### 8.1 用户消息缓冲

第一种缓冲是用户程序中声明的变量或数组。

例如：

```c
send(A, 16, Q, tag)
```

这里的 `A` 就是用户缓冲。

------

### 8.2 系统消息缓冲

第二种缓冲由消息传递系统创建和管理。

它用于发送消息时暂存数据，用户程序中不直接出现。

------

### 8.3 用户级临时缓冲

第三种是 MPI 允许用户划出一块内存，作为任意消息的中间缓冲。

课件图示中比较了：

```text
只使用用户缓冲
使用系统缓冲 S
使用用户级临时缓冲 T
```

------

## 9. MPI 中的消息信封

### 9.1 消息信封三要素

MPI 发送例程中的信封由三项组成：

```text
destination
tag
communicator
```

其中：

```text
destination：目的进程编号
tag：消息标签
communicator：通信子
```

------

### 9.2 为什么需要 tag

课件给了一个例子。

如果进程 P 连续发送两个消息：

```text
send(A, 32, Q)
send(B, 16, Q)
```

进程 Q 连续接收：

```text
recv(X, 32, P)
recv(Y, 16, P)
```

如果没有标签，后发送的 `B` 可能先到达，就可能被第一个 `recv` 错误接收到 `X` 中。

使用标签后：

```text
send(A, 32, Q, tag1)
send(B, 16, Q, tag2)

recv(X, 32, P, tag1)
recv(Y, 16, P, tag2)
```

就能避免消息类型混淆。

------

### 9.3 tag 与服务请求

另一个例子中，两个客户进程 P 和 R 向服务进程 Q 发送请求。

如果使用不同标签：

```text
tag1 表示一种服务请求
tag2 表示另一种服务请求
```

服务进程可以通过 `Status.Tag` 判断消息类型，从而按不同方式处理。

------

## 10. 通信子 communicator

### 10.1 通信子的定义

课件定义：

```text
通信子 = 进程组 + 上下文
```

进程组是有限有序进程集合，大小为 `group_size`，进程编号为：

```text
0, 1, ..., n-1
```

常用查询函数：

```c
MPI_Comm_size(communicator, &group_size);
MPI_Comm_rank(communicator, &my_rank);
```

------

### 10.2 为什么需要通信子

仅靠 tag 不够安全。

因为不同库函数或用户代码可能不小心使用相同 tag。
通信子通过不同上下文隔离通信，即使两个通信子包含相同进程组，也不会混淆消息。

这对并行库开发特别重要。

------

### 10.3 MPI_Comm_dup

```c
MPI_Comm_dup(MPI_COMM_WORLD, &MyWorld)
```

创建一个新的通信子 `MyWorld`。

它包含和 `MPI_COMM_WORLD` 相同的进程组，但有不同上下文。

------

### 10.4 MPI_Comm_split

课件给出 10 个进程的例子：

```c
Color = my_rank % 3;
Key = my_rank / 3;
MPI_Comm_split(MyWorld, Color, Key, &SplitWorld);
```

含义是：

```text
按 Color 把原通信子拆成多个子通信子
按 Key 决定新通信子中的 rank 顺序
```

这可以把全体进程划分成多个通信组。

------

## 11. MPI_Recv 和状态信息

### 11.1 MPI_Recv 参数

形式是：

```c
MPI_Recv(address, count, datatype, source, tag, communicator, status)
```

其中：

```text
source 和 tag 可以使用通配符
MPI_ANY_SOURCE
MPI_ANY_TAG
```

------

### 11.2 Status 的作用

接收完成后，`MPI_Status` 中保存：

```text
Status.MPI_SOURCE：实际源进程编号
Status.MPI_TAG：实际消息标签
```

实际收到的数据项数可以通过：

```c
MPI_Get_count(&Status, MPI_INT, &C)
```

得到。

------

### 11.3 Status 适用场景

当一个服务进程可能从不同源进程接收不同大小、不同标签的消息时，Status 很有用。

典型模式是：

```c
MPI_Recv(..., MPI_ANY_SOURCE, MPI_ANY_TAG, comm, &Status);

switch (Status.MPI_TAG) {
    case tag_0: ...
    case tag_1: ...
}
```

------

## 12. MPI 中的四种通信模式

### 12.1 同步发送 synchronous

同步发送要求相应接收已经启动后发送才返回。

它不需要接收端额外缓冲，但等待较强。

MPI 对应函数：

```text
MPI_Ssend
MPI_Issend
```

------

### 12.2 缓冲发送 buffered

缓冲发送要求用户先提供缓冲区：

```c
MPI_Buffer_attach(buffer, size)
```

结束时可以：

```c
MPI_Buffer_detach(...)
```

对应函数：

```text
MPI_Bsend
MPI_Ibsend
```

------

### 12.3 标准发送 standard

标准发送可能是同步的，也可能是缓冲的，取决于 MPI 实现。

对应函数：

```text
MPI_Send
MPI_Isend
```

------

### 12.4 就绪发送 ready

就绪发送要求用户确信相应接收已经开始。

它可以使用更高效协议，但如果接收没有开始就调用，就可能出错。

对应函数：

```text
MPI_Rsend
MPI_Irsend
```

------

## 13. 点对点通信

### 13.1 MPI 点对点函数表

课件列出：

| 类型             | 阻塞      | 非阻塞     |
| ---------------- | --------- | ---------- |
| Standard Send    | MPI_Send  | MPI_Isend  |
| Synchronous Send | MPI_Ssend | MPI_Issend |
| Buffered Send    | MPI_Bsend | MPI_Ibsend |
| Ready Send       | MPI_Rsend | MPI_Irsend |
| Receive          | MPI_Recv  | MPI_Irecv  |
| Completion Check | MPI_Wait  | MPI_Test   |

------

### 13.2 进程流水线例子

课件给出三进程流水线：

```text
W → P → X → Q → Y → R → Z
```

其中进程 Q 逻辑是：

```text
接收 X
计算 Y = Q(X)
发送 Y
```

为了重叠通信和计算，可以使用非阻塞接收和非阻塞发送：

```c
MPI_Irecv(...);
MPI_Isend(...);
CurrentY = Q(CurrentX);
```

------

### 13.3 双缓冲

为了避免正在发送的数据被修改，课件给出双缓冲思想：

```text
Xbuf0 / Xbuf1
Ybuf0 / Ybuf1
```

一组缓冲用于当前计算，另一组缓冲用于通信。
通过 `MPI_Wait` 确认通信完成后再复用缓冲。

------

### 13.4 非阻塞通信中的错误例子

课件给出一个可能死锁或结果不确定的例子。

问题在于：

```text
MPI_Issend 是非阻塞同步发送
MPI_Irecv 可能由于条件不满足而没有启动
```

可能结果包括：

```text
死锁
打印 Y=0
打印 Y=5
```

这说明非阻塞通信虽然灵活，但必须认真处理完成检查和数据依赖。

## 14. MPI 群集通信

### 14.1 Broadcast 广播

形式：

```c
MPI_Bcast(Address, Count, Datatype, Root, Comm)
```

Root 进程把相同消息发送给通信子中的所有进程。

------

### 14.2 Scatter 播撒

形式：

```c
MPI_Scatter(SendAddress, SendCount, SendDatatype,
            RecvAddress, RecvCount, RecvDatatype,
            Root, Comm)
```

Root 给每个进程发送不同的数据块。

------

### 14.3 Gather 聚集

形式：

```c
MPI_Gather(SendAddress, SendCount, SendDatatype,
           RecvAddress, RecvCount, RecvDatatype,
           Root, Comm)
```

Root 从每个进程收集一块数据，并按 rank 顺序放入接收缓冲。

------

### 14.4 Allgather 全局收集

形式：

```c
MPI_Allgather(...)
```

每个进程都收集到所有进程的数据。

它可以理解为：

```text
每个进程都执行一次 gather 的效果
```

------

### 14.5 Alltoall 全局交换

形式：

```c
MPI_Alltoall(...)
```

每个进程都向所有进程发送一个消息，同时也从所有进程接收一个消息。

课件指出，一次 all-to-all 中共有：

```text
n² 个消息通信
```

这是通信量很大的模式。

------

### 14.6 Reduction 归约

形式：

```c
MPI_Reduce(SendAddress, RecvAddress, Count, Datatype,
           Op, Root, Comm)
```

所有进程贡献局部值，通过 `Op` 归约成一个结果，存入 Root。

常见操作：

```text
求和
求最大值
求最小值
乘积
```

------

### 14.7 Scan 扫描

形式：

```c
MPI_Scan(SendAddress, RecvAddress, Count, Datatype, Op, Comm)
```

scan 不需要 Root，因为它会为每个进程生成一个前缀结果。

例如 rank 0 到 rank k 的值按某个操作组合后，结果存入 rank k。

------

### 14.8 Barrier 路障

形式：

```c
MPI_Barrier(Comm)
```

通信子中的所有进程互相等待，直到所有进程都调用了 `MPI_Barrier` 才能继续。

------

### 14.9 群集通信共同特点

课件总结了群集通信的几个关键规则：

```text
通信子中的所有进程必须调用群集通信例程
Count 和 Datatype 在相关进程中必须一致
群集例程中没有 tag 参数
MPI-1 只支持阻塞的通信子内群集通信
```

如果只有部分进程调用群集通信，而其他进程没有调用，行为是不确定的，可能死锁，也可能得到错误结果。

## 15. MPI 扩展

### 15.1 MPI-2 对 MPI-1 的扩展

课件列出 MPI-2 的扩展包括：

```text
动态进程
单边通信
非阻塞群集通信
通信子间群集通信
MPI-IO
Fortran90 和 C++ 绑定
实时处理支持
面向 profiling、monitoring、debugging 的外部接口
```

------

### 15.2 动态进程

MPI-1 不定义如何创建进程，必须依赖平台。

MPI-2 支持动态进程，优点包括：

```text
可移植地创建进程
有利于 PVM 程序移植到 MPI
支持 client/server 和 process farm
更有效利用资源和负载平衡
支持一定容错能力
```

------

### 15.3 MPI_Spawn

课件给出形式：

```c
MPI_Spawn(command_line, minprocs, maxprocs, info,
          root, comm, intercomm, array_of_errcodes)
```

它尝试派生子进程，并通过 `intercomm` 建立父进程组和子进程组之间的通信。

------

### 15.4 单边通信 RMA

MPI-2 引入远程存储访问：

```text
RMA = Remote Memory Access
```

单边通信允许一个进程把数据放到另一个进程，或从另一个进程取数据，而不需要另一边显式参与对应的 send/receive。

相关函数：

```text
MPI_Get
MPI_Put
```

课件强调，MPI-1 中点对点通信都是双边通信，发送者和接收者都必须参与。

## 16. MPI 计算 π 的例子

### 16.1 计算思想

仍然使用积分公式：

```text
π = ∫_0^1 4 / (1+x²) dx
```

MPI 程序中：

```text
每个进程计算一部分区间
每个进程得到局部 mypi
通过 MPI_Reduce 求总和 pi
```

------

### 16.2 关键 MPI 操作

程序中使用：

```c
MPI_Bcast(&n, 1, MPI_LONG, 0, MPI_COMM_WORLD);
```

把区间数 `n` 从 0 号进程广播给所有进程。

然后：

```c
MPI_Reduce(&mypi, &pi, 1, MPI_DOUBLE,
           MPI_SUM, 0, MPI_COMM_WORLD);
```

把所有进程的局部积分结果求和到 0 号进程。

课件给出：


其中 `-np 2` 表示启动 2 个 MPI 进程。

## 26. HPF 的 FORALL 机制

### 26.1 简单 FORALL

例子：

```fortran
FORALL (K=1, M-1) X(K+1)=X(K)
```

含义是对所有满足条件的 `K` 同时执行赋值。

另一个例子：

```fortran
FORALL (I=1:N, J=1:N) X(I,J)=Y(J,I)
```

这表示把 `Y` 的转置赋给 `X`。

------

### 26.2 带限制的 FORALL

例子：

```fortran
FORALL (i=2:5, X(i)>0) X(i)=X(i-1)+X(i+1)
```

这里只有满足 `X(i)>0` 的活动索引才执行赋值。

课件示例中，如果初始：

```text
X = {1, -1, 2, -2, 3, -3}
```

活动索引是：

```text
{3, 5}
```

因此只更新 `X(3)` 和 `X(5)`。

------

### 26.3 多个 for-三元组

例子：

```fortran
FORALL(i=1:2, j=1:3, Y(i,j)>0)
    Z(i,j)=1/Y(i,j)
```

这里索引是二维联合索引：

```text
(i,j)
```

有效集合为：

```text
{(1,1),(1,2),(1,3),(2,1),(2,2),(2,3)}
```

再从中选出满足 `Y(i,j)>0` 的活动集合。

------

### 26.4 FORALL 结构

FORALL 结构可以包含多条语句：

```fortran
FORALL (I=2:9)
    A(I) = A(I-1) + A(I+1)
    B(I) = A(I)
ENDFORALL
```

课件强调执行语义：

```text
先对所有 I 求右端表达式
再整体赋给 A
然后再把新 A 的值赋给 B
```

这体现了 FORALL 的并行赋值语义。

## 27. HPF 数据映射

### 27.1 数据映射的定义

数据映射就是：

```text
把数据分布到处理器上
```

在分布存储系统上，必须先对数组划分，再分配到处理器，计算才能在各处理器上进行。

数据映射目标：

```text
最小化通信开销
负载均匀分布
```

------

### 27.2 Owner-compute 规则

课件提到 HPF 编译器可以使用 owner-compute 规则：

```text
数据在哪个处理器上，就在哪个处理器上执行相关计算
```

因此数据分布会直接决定计算负载分布。

------

### 27.3 PROCESSORS

`PROCESSORS` 用于描述虚拟处理器结构。

例子：

```fortran
!HPF$ PROCESSORS P4(4)
```

表示 4 个虚拟处理器的一维结构。

```fortran
!HPF$ PROCESSORS P22(2,2)
```

表示 2×2 的二维虚拟处理器结构。

------

### 27.4 ALIGN

`ALIGN` 用于描述数据对准。

例子：

```fortran
!HPF$ ALIGN A(I) WITH B(I)
```

表示 `A(I)` 和 `B(I)` 分配到同一个处理器。

```fortran
!HPF$ ALIGN A(I,J) WITH B(J,I)
```

表示 `A(I,J)` 和 `B(J,I)` 对准，适合转置关系。

------

### 27.5 不同维数组对准

例子：

```fortran
!HPF$ ALIGN A(I,*) WITH B(I)
```

表示 `A` 的第 `I` 行所有元素与 `B(I)` 对准。

```fortran
!HPF$ ALIGN A(I) WITH B(I,*)
```

表示 `A(I)` 被重复对准到 `B` 第 `I` 行各元素所在处理器。

------

### 27.6 DISTRIBUTE

`DISTRIBUTE` 用于描述数据分布。

主要方式：

```text
BLOCK
CYCLIC
```

------

### 27.7 BLOCK 分布

例子：

```fortran
!HPF$ DISTRIBUTE A(BLOCK) ONTO P4
```

把数组 `A` 分成 4 个连续块，分配给 4 个处理器。

```fortran
!HPF$ DISTRIBUTE A(BLOCK(30)) ONTO P4
```

表示每块 30 个元素，最后一块可能不足 30。

------

### 27.8 二维 BLOCK 分布

例子：

```fortran
!HPF$ DISTRIBUTE B(BLOCK, BLOCK) ONTO P22
```

表示二维数组 `B` 在行、列两个方向都分块，分到 2×2 处理器结构上。

```fortran
!HPF$ DISTRIBUTE B(*, BLOCK) ONTO P4
```

表示行方向不分布，只在列方向分块。

------

### 27.9 CYCLIC 分布

例子：

```fortran
!HPF$ DISTRIBUTE A(CYCLIC) ONTO P4
```

表示从 `A(1)` 开始，把元素循环分配给 4 个处理器。

```fortran
!HPF$ DISTRIBUTE A(CYCLIC(2)) ONTO P4
```

表示每两个元素一组，循环分配。

例如：

```text
处理器1：A(1), A(2), A(9), A(10), ...
处理器2：A(3), A(4), A(11), A(12), ...
```

------

### 27.10 数据映射例子

课件给出代码：

```fortran
integer A(100), B(100), C(101), i
!HPF$ ALIGN A(i) WITH B(i-1)
!HPF$ PROCESSOR N(4)
!HPF$ DISTRIBUTE A(BLOCK) ONTO N
!HPF$ DISTRIBUTE C(CYCLIC) ONTO N
FORALL (i=2:100)
    A(i) = A(i)+B(i-1)
    C(i) = C(i-1)+C(i)+C(i+1)
ENDFORALL
```

含义是：

```text
A 和 B 根据下标偏移进行对准
A 用 BLOCK 分布
C 用 CYCLIC 分布
FORALL 表达并行数组计算
```

课件第 121 页图把这个过程分成：

```text
ALIGN：逻辑对准
DISTRIBUTE：映射到虚拟处理器
Physical Mapping：映射到实际机器
```

## 28. 高斯消去法的 HPF 程序

### 28.1 程序结构

课件最后给出一个 HPF 高斯消去法程序。

关键声明：

```fortran
parameter n=32
real A(n,n+1), x(n)
!HPF$ PROCESSOR Nodes(4)
!HPF$ ALIGN x(i) WITH A(i,j)
!HPF$ DISTRIBUTE A(BLOCK,*) ONTO Nodes
```

含义是：

```text
定义4个虚拟处理器
x(i) 与 A(i,j) 对准
A 的第一维按 BLOCK 分布
A 的第二维不分布
```

------

### 28.2 消元阶段

核心循环：

```fortran
do i=1,n-1
    pivot_location = MAXLOC(ABS(A(i:n,i)))
    swap(...)
    A(i,i:n+1)=A(i,i:n+1)/A(i,i)
    FORALL (j=i+1:n,k=i+1:n+1)
        A(j,k)=A(j,k)-A(j,i)*A(i,k)
end do
```

其中：

```text
MAXLOC 用于选主元
swap 交换主元行
FORALL 并行更新剩余矩阵区域
```

这体现了 HPF 在稠密线性代数中的数据并行写法。

------

### 28.3 回代阶段

回代代码：

```fortran
do i=n,1,-1
    x(i)=A(i,n+1)
    A(1:i-1,n+1)=A(1:i-1,n+1)-A(1:i-1,i)*x(i)
end do
```

每求出一个 `x(i)`，就更新前面方程的右端项。

------

### 28.4 数据分布

课件最后说明，当 `n=32` 且有 4 个节点时：

```text
结点1：A(1:8, 1:33), X(1:8)
结点2：A(9:16, 1:33), X(9:16)
结点3：A(17:24, 1:33), X(17:24)
结点4：A(25:32, 1:33), X(25:32)
```

这说明 `A` 的行被分块分给四个节点。

## 29. 本章三种编程方式对比

| 方式           | 代表 | 核心思想                    | 优点                           | 缺点                     |
| -------------- | ---- | --------------------------- | ------------------------------ | ------------------------ |
| 消息传递       | MPI  | 显式 send/receive           | 控制力强、性能好、适合分布存储 | 程序员负担大             |
| 虚拟机消息传递 | PVM  | 把异构主机组成虚拟并行机    | 动态灵活、自包含系统           | 非标准，行为可能非确定   |
| 数据并行       | HPF  | 全局数组操作 + 数据分布指令 | 编程抽象高、减轻通信负担       | 依赖编译器和良好数据分布 |

## 30. 本章核心总结

第十四章的主线是：

```text
分布存储系统中，每个进程只能访问本地数据；
要实现并行计算，必须解决数据如何分布、消息如何传递、进程如何同步的问题。
```

### 30.1 MPI 的核心

```text
MPI = 标准化、可移植、高效的消息传递接口
```

重点掌握：

```text
MPI_Init / Finalize
MPI_Comm_size / rank
MPI_Send / Recv
datatype
tag
communicator
Status
Bcast / Scatter / Gather / Alltoall / Reduce / Scan / Barrier
```

------

### 30.2 PVM 的核心

```text
PVM = 把多台主机组织成并行虚拟机
```

重点掌握：

```text
pvmd
host_file
pvm console
pvm_spawn
pvm_send / recv
pvm group
pvm_reduce
```

------

### 30.3 HPF 的核心

```text
HPF = 用 Fortran 风格的数据并行机制描述分布存储计算
```

重点掌握：

```text
FORALL
PROCESSORS
ALIGN
DISTRIBUTE
BLOCK
CYCLIC
owner-compute
```

## 31. 复习重点

这一章最应该掌握：

```text
1. 消息传递模型中进程只能访问私有数据
2. MPI 和 PVM 是最重要的消息传递系统
3. 同步、阻塞、非阻塞三种消息传递方式的区别
4. 非阻塞通信可以重叠计算，但需要 wait/test 检查
5. MPI 消息 = 消息缓冲 + 消息信封
6. MPI_Send 六个参数的含义
7. tag 用于区分消息类型，communicator 用于隔离通信上下文
8. MPI derived datatype 用于描述非连续或混合类型消息
9. MPI 四种发送模式：standard、synchronous、buffered、ready
10. MPI 点对点通信和群集通信函数
11. MPI-2 的动态进程、RMA、MPI-IO
12. PVM 的 pvmd、host、task、动态主机配置和动态分组
13. 数据并行模型的特点：全局命名空间、隐式通信、显式数据分布
14. HPF 的 FORALL、ALIGN、DISTRIBUTE、BLOCK、CYCLIC
15. HPF 高斯消去法中数据按行块分布，FORALL 用于并行更新
```

一句话概括：

> 第十四章讲的是分布存储系统怎样写并行程序：MPI 和 PVM 代表显式消息传递，程序员直接控制通信；HPF 代表数据并行，让程序员通过数组操作和数据分布指令表达并行性，由编译器生成通信。
